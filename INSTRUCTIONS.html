Animated intros
in under 5 minutes.

This guide teaches you — even with zero coding experience — how to turn a raw Peter Holmquist talking-head clip into a branded, word-synced animated intro ready to drop into Premiere. You'll be doing real motion graphics work by chatting with an AI assistant in plain English. No timelines, no keyframes, no Adobe After Effects.

1What you'll be doing

Your job in one sentence: get a raw Peter clip, give it to Claude, review the result, deliver the PNG sequence to Tim.

Reference output — this is an example of what you'll be producing. 17 seconds, 1920×1080, DPE brand colors, word-synced text callouts, full-screen list moment when Peter enumerates pain points.

The full flow

① Tim sends you a clip
Raw Peter talking-head video, usually 15-60 seconds. Delivered via Google Drive or Dropbox.
② You run /dpe-intro
Open Claude Code, type the command with the clip's path. Claude handles transcription, animation design, and rendering.
③ You review the preview
Claude produces a preview MP4 with Peter + the animated overlay. Watch it. Approve or request tweaks.
④ You deliver the PNG sequence
Zip the folder of PNG frames, upload to the shared drive, message Tim with the link.

2One-time setup

Do this once, the first time you join the team. It takes about 15 minutes. After this, every new intro takes about 5 minutes.

!
You need a Windows, Mac, or Linux computer
~8 GB free disk space, stable internet, and the ability to install software. That's it.
  1. Install Claude Code

    Claude Code is the AI assistant you'll talk to. Download and install from:

    https://claude.com/claude-code

    Follow the installer. On first launch, it will ask you to log in with your Anthropic account. (Tim will set up your account or cover the subscription.)

  2. Install Node.js

    Node.js is the runtime that Hyperframes (the video renderer) and the transcription script run on. Go to:

    https://nodejs.org/en/download

    Download the LTS version (large green button). Run the installer with default options. Restart your terminal / Claude Code after install.

    To verify, open Claude Code and type in the terminal:

    node --version

    You should see something like v22.x.x or higher.

  3. Install FFmpeg

    FFmpeg is the video/audio processor. How you install depends on your operating system:

    • Windows: Open PowerShell as Administrator and run winget install Gyan.FFmpeg. After install, close and reopen PowerShell.
    • Mac: Install Homebrew if you haven't (brew.sh), then brew install ffmpeg
    • Linux: sudo apt install ffmpeg (or equivalent for your distro)

    Verify with:

    ffmpeg -version
  4. Get the DPE intro toolkit

    Clone this repository. Open a terminal in a folder where you keep your projects (e.g. C:\Projects on Windows, or ~/Projects on Mac/Linux), then run:

    git clone https://github.com/gebhardtim1-lang/dpe-youtube-intro.git
cd dpe-youtube-intro

    If you don't have git, download it from git-scm.com, or just download the repo as a ZIP from GitHub and unzip it anywhere you like.

  5. Get an OpenAI API key

    Needed for transcription. Roughly $0.006 per minute of audio — about $0.002 per 15-second intro (basically free).

    Go to platform.openai.com/api-keys, sign in (or sign up), click "Create new secret key", copy the key it shows you.

    Inside your dpe-youtube-intro folder, make a copy of the file .env.example and rename the copy to .env. Open it in any text editor and replace the placeholder:

    OPENAI_API_KEY=sk-paste-your-real-key-here

    Save. Done.

    !
    Never share this key
    Anyone with the key can spend money on your OpenAI account. Keep .env out of emails, Slack, and screen shares. Git already ignores it by default.
  6. Run the install skill

    Open Claude Code inside the dpe-youtube-intro folder. In the chat, type:

    /dpe-intro-install

    Claude will:

    • Double-check Node.js and FFmpeg are installed correctly
    • Install Hyperframes (the video renderer) — just say "yes" to any prompts
    • Install the npm packages the transcribe script needs
    • Scaffold a working/ folder you'll use per video
    • Run a quick smoke test to confirm everything works

    If anything fails, Claude will tell you how to fix it. Just read the message and follow instructions.

✓
You're done with setup
You'll never have to do this again. Every new intro from now on uses the per-video workflow below.

3Per-video workflow

This is what you'll do every time Tim sends a new Peter clip. Target time: 5 minutes.

  1. Get the raw clip from Tim

    Tim sends you a link to a Google Drive / Dropbox file. Download it. Save to any folder you want — e.g. C:\DPE-Intros\raw\.

    Name doesn't matter, but something memorable helps (e.g. 2026-04-22-peter-5m-pitch.mp4).

  2. Open Claude Code in the dpe-youtube-intro folder

    Open the dpe-youtube-intro folder in Claude Code. (Usually: File → Open Folder → pick it.)

  3. Ask Claude to make the intro

    In the chat, type:

    /dpe-intro "C:\path\to\your\raw-clip.mp4"

    Use the real path to the clip you downloaded. Keep the quotes if the path has spaces.

    Claude will now:

    • Transcribe the clip with word-level timing (~30 seconds)
    • Read the brand guide and the reference pattern
    • Pick 3-5 punchy phrases from what Peter says
    • Detect any listing moment (3+ items being enumerated) — if found, design a full-screen list takeover
    • Generate the HTML composition with everything word-synced
    • Render a preview MP4 (composited: Peter + animation)

    Total time: 2-3 minutes while it works.

  4. Review the preview

    Claude will give you a file path like:

    outputs/your-clip-preview.mp4

    Open it and watch. Check:

    • Do the phrases appear exactly when Peter says the matching word?
    • Is the timing comfortable — nothing too short to read?
    • Do animations feel DPE-branded (cyan + gold accents, blueprint grid, bold Montserrat)?
    • Is Peter's face ever covered by a phrase? (shouldn't be)
  5. Approve, or request changes

    If it looks good, just say "approved, render the final". Claude renders the overlay-only MOV + PNG sequence.

    If it needs changes, type what's off. Examples of feedback that works:

    • "The '$5 Million' phrase appears too late — should land when he says '5'"
    • "Use 'Earn more' instead of '$5 Million' as the hero phrase"
    • "No full-screen list — keep it phrases-only for this one"
    • "Make it 12 seconds instead of 17"

    Claude iterates. Preview again. Approve.

  6. Get your final files

    After approval, Claude tells you the output paths. You'll have three files per intro:

    Preview MP4
    Peter + overlay composited. For reference only. Don't deliver this.
    PNG sequence folder
    This is what you deliver to Tim. ~500 PNG files with transparency. Zip the folder before uploading.
    Alpha MOV
    Alternative single-file with transparency. If Tim prefers MOV over PNG, deliver this instead.

4Delivery

How to hand the finished intro to Tim.

  1. Zip the PNG sequence folder

    The PNG sequence is a folder named something like your-clip-png-sequence/ containing ~500 PNG files. Right-click it → Compress / Send to → Compressed folder. You'll get a .zip file about 1-2 GB.

  2. Upload to the shared drive

    Tim will tell you the shared folder (Google Drive or Dropbox). Upload the .zip there. Also upload the -preview.mp4 — Tim uses it to verify without having to unzip.

  3. Message Tim

    In Slack or WhatsApp, send:

    Done with <clip-name> — preview + PNG sequence uploaded to <folder link>. Duration: X seconds. Let me know if anything needs to change.

5Troubleshooting

If something breaks, check here first.

Problem Fix
"node is not recognized" or similar Node.js isn't installed yet, or you didn't restart your terminal after installing it. Close Claude Code completely and reopen it.
"ffmpeg is not recognized" Same as above but for FFmpeg. On Windows, you may need to add it to your PATH manually if winget didn't do it.
"OPENAI_API_KEY not set" Your .env file is missing or the key is still the placeholder. Open .env and paste your real key.
Claude says "Hyperframes not installed" Re-run /dpe-intro-install and let it retry the Hyperframes install step.
Rendering takes more than 5 minutes Your computer might be low on RAM. Close other heavy apps. Tell Claude to render with "draft quality" first, which is faster.
The preview looks wrong — fonts missing, text overflowing Usually a composition issue. Send Claude a screenshot of what's wrong and say "the preview has <issue> — fix it and re-render."
Phrases don't sync with Peter's words This sometimes happens if the clip has a silent lead-in or mid-sentence pause. Tell Claude: "there's a 2-second silence at the start — trim it and regenerate."
"Invalid OpenAI API key" The key you pasted is wrong or disabled. Regenerate a new key at platform.openai.com/api-keys and update .env.
The PNG sequence zip is too big to upload Use Dropbox or Google Drive web uploader (not a free file-sharing site). 1-2 GB is normal — 525 PNG frames at 1920×1080 with alpha is big. Alternative: send the MOV with alpha instead; that's ~1 GB single file.
